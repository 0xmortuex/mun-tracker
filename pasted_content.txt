PRD: MUN TR — Turkish Model UN Conference TrackerSummary
A static, mobile-first website that displays Model UN conferences happening in Turkey, with filters and search. Audience: Turkish high school students (especially beginners) looking for conferences to attend. Built to be deployed on GitHub Pages with no backend, no build step, no framework.Owner
0xmortuex (Fadi Raad) — 9th-grade student in Istanbul. Public personal project.Problem
Existing MUN listing sites (mymun.com, munturkey.com) have three problems:

Global scope — Turkey-specific filtering is buried
No personalization for high school students vs. university delegates
No Turkish Lira fee display alongside USD
Hard to share specific filtered views
Goal
A single shareable URL where a Turkish high schooler can:

See all Turkey MUN conferences in one place
Filter by location, fee, level, language, status, beginner-friendliness, date range
Search by name/acronym
Click a card to see full details
Share filtered views with friends via URL hash
Non-goals

No user accounts, login, or authentication
No notifications or email
No backend, no API, no database — static only
No conference applications submitted through this site (links go to official conference sites)
No conference reviews, ratings, or comments
No middle school (JMUN) or university-only conferences
No data analytics or tracking
Design BriefTheme
Dark academic. Think Oxford / classical institution, not tech startup.
Background: near-black #0a0a0a
Surface (cards, filter bar): elevated charcoal #131316 or #111114
Borders: subtle #27272a
Primary text: off-white #fafafa
Secondary text: muted gray #a1a1aa
Accent color: Oxford Blue #002147 — used for active states, badges, CTAs
Accent hover/lighter: #0a3a8a (lifted Oxford blue)
Status colors (dots only, not large blocks):

Open: green #22c55e
Past/completed: muted gray #71717a
Postponed: orange #f59e0b
Dormant: dim red #dc2626


Typography

UI: Inter (Google Fonts) — body, headings, labels
Numbers/dates: JetBrains Mono (Google Fonts) — mono font for visual contrast
Sizes: 14px body / 12px secondary / 24px card titles / 32px page title
Layout

Header: Site title "MUN TR" + tagline "Turkish Model UN conferences, tracked"
Sub-header: last-updated date + disclaimer "Community-sourced data — verify before applying"
Search bar (sticky on scroll desktop): full-width input, placeholder "Search conferences by name or acronym...", searches both name and acronym fields
Filter bar (sticky on scroll desktop, collapsible on mobile behind "Filters (N active)" button):

Location: All / Istanbul / Other Turkey
Status: All / Open / Past / Postponed / Dormant
Date range: All / Next 30 days / Next 60 days / Next 90 days
Fee range: All / Free / Under ₺2000 / ₺2000+
Level: All / High school only / HS + University
Language: All / English only / English + other languages
Beginner-friendly: toggle
Reset filters: button


Sort dropdown: Status priority (default) / Date ascending / Fee ascending / Alphabetical
Card grid: 3 columns desktop / 2 tablet / 1 mobile
Each card:

Top: large bold acronym + status dot with text
Conference name (smaller, muted)
Host school + city
Dates (mono font) or "TBA" if unknown
Fee in TRY + USD if known, else "Not disclosed"
Beginner-friendly pill if applicable (Oxford blue accent)
Edition label (e.g., "5th edition")
Hover: lift 2px with subtle shadow
Click: expand inline to show full details


Expanded card details:

Languages
Rules of procedure
Committees (if known)
Character/description
Official website link
Instagram link
Email contact
"Verify before applying" italic disclaimer


Past/completed conferences: auto-collapsed by default behind "Show past conferences (N)" toggle — saves vertical space, focus stays on actionable items
Empty state when no matches: "No conferences match your filters" + Reset button
Counter: "Showing X of N conferences"
Footer: Author credit + last-updated + repo link + data disclaimer
Behavior

Vanilla JavaScript only — no React, no Vue, no framework
Single index.html, style.css, app.js, conferences.json
Filters and search update the visible cards in real-time (no page reload)
All filter state reflected in URL hash so people can share filtered views (e.g., #search=munb&status=open)
URL hash updates as filters change; URL is the source of truth on page load
Accessibility: semantic HTML (article, nav, header), keyboard-navigable filters, WCAG AA color contrast
Mobile: hamburger or accordion collapse for filter bar at <768px
No localStorage, no sessionStorage — all state in URL hash and in-memory
No analytics, tracking, or external scripts beyond Google Fonts CDN
